## Global Constraints

Every task's requirements implicitly include this section. Copy exact values verbatim.

- **Oracle governs over the spec.** Where the Phase 1 spec's guessed shape differs from the vendored oracle (`apps/api/tests/compat/oracle/openapi-final.yaml`, info.version 3.0.0, 84 ops), implement the **oracle** shape and freeze against it. (Known case: `POST /v2/list-agents` — see Task 7.)
- **Oracle omits null optionals.** The oracle marks optional fields non-nullable+optional (not `nullable:true`); serializing `null` is non-conformant. Call/agent routes already use `response_model_exclude_none=True` (calls) / `.model_dump(exclude_none=True)` (agents). **New routes must follow the same pattern.** EXCEPTION: a conformance test that checks a *required* field is present must serialize WITHOUT `exclude_none` so a missing-required violation surfaces (Task 1).
- **`V2CallBase` requires `call_id, agent_id, agent_version, call_status`** (non-nullable). `V2PhoneCallResponse` = `allOf[{required: call_type, from_number, to_number, direction}, V2CallBase]`. Any successfully-created phone call MUST carry a non-null `agent_id` and `agent_version`.
- **Compat routers have NO path prefix** (`router = APIRouter(tags=["compat-calls"])` at `calls.py:39`, `tags=["compat-agents"]` at `agents.py:35`). Routes use absolute paths in the decorator (e.g. `"/v2/list-agents"`). The sub-app is mounted at `/`.
- **Every new route MUST call `_audit(request, "<op-name>", <resource_id>)`** before returning (`calls.py:42-44`, `agents.py:38-40`).
- **ID codecs:** `ids.decode_agent_id` (validates `agent_` prefix → `CompatError(422)`), `ids.decode_call_id`, `ids.decode_llm_id` (`llm_` prefix). Use the right decoder.
- **Errors** are raised as `CompatError(<status>, "<message>")` (no PHI in messages).
- **RLS / tenancy:** all DB access goes through `get_compat_db` (org-scoped session). Do not bypass it.
- **Commit format:** `type(scope): description` — scope is `api` for everything except Task 9 (`agent`). Attribution trailers are disabled globally — do NOT add `Co-Authored-By` or a tool footer.
- **Gate before every commit:** `cd apps/api && uv run ruff check . && uv run ruff format --check . && uv run mypy && uv run pytest -q` (agent task: `cd services/agent && uv run ruff check . && uv run ruff format --check . && uv run mypy && uv run pytest -v`). CI also runs mypy — do not skip it.

```python
import json
import uuid

DYNAMIC_VARS_TOPIC = "usan/vars"


async def force_hangup(room: str, settings: Settings) -> None:
    """Abruptly disconnect every participant (incl. the SIP/PSTN leg) in a live room.

    Best-effort: a missing room or transient LiveKit error is logged, never raised — the
    caller's primary outcome (marking the call cancelled) must not be masked.
    """
    try:
        async with build_livekit_api(settings) as lkapi:
            await lkapi.room.delete_room(api.DeleteRoomRequest(room=room))
    except Exception:
        logger.bind(room=room).warning("force_hangup failed (best-effort)")


async def send_dynamic_vars(room: str, variables: dict[str, str], settings: Settings) -> None:
    """Broadcast a dynamic-variable update to a live room over the data plane.

    Wire contract (mirrored in services/agent dynamic_vars.py): topic 'usan/vars', RELIABLE,
    JSON object of str->str. Best-effort; never raises.
    """
    try:
        payload = json.dumps(variables).encode()
        async with build_livekit_api(settings) as lkapi:
            await lkapi.room.send_data(
                api.SendDataRequest(
                    room=room,
                    data=payload,
                    kind=api.DataPacket.Kind.RELIABLE,
                    topic=DYNAMIC_VARS_TOPIC,
                    nonce=uuid.uuid4().bytes,
                )
            )
    except Exception:
        logger.bind(room=room).warning("send_dynamic_vars failed (best-effort)")
```

```python
"""Agent-side receiver for mid-call dynamic-variable updates.

Wire contract (mirror of apps/api livekit_dispatch.py): LiveKit data-plane topic 'usan/vars',
RELIABLE, JSON object of str->str. On receipt, merge into the running vars, re-substitute the
instruction template, and hand the new instructions back to the caller (which updates the agent).
"""

from __future__ import annotations

import json
from collections.abc import Callable, Mapping

from loguru import logger

from usan_agent import prompt_vars

DYNAMIC_VARS_TOPIC = "usan/vars"


def apply_dynamic_vars(
    payload: bytes,
    *,
    current_vars: Mapping[str, str],
    template: str,
    on_update: Callable[[str], None],
) -> None:
    if not payload:
        return
    try:
        incoming = json.loads(payload.decode())
    except (ValueError, UnicodeDecodeError):
        logger.warning("dropped malformed dynamic-vars data packet")
        return
    if not isinstance(incoming, dict) or not incoming:
        return
    merged = {**current_vars, **{str(k): str(v) for k, v in incoming.items()}}
    on_update(prompt_vars.substitute(template, merged))
```
